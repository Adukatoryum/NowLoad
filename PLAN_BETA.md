## Крок 1 — Загрузіць праект на GitHub (прыватны)

**Чаму GitHub?** Захоўвае ўсе версіі. Калі нешта зломіцца — можна вярнуцца назад.

### Зрабіць адзін раз:

1. Зарэгіструйся на [github.com](https://github.com) (калі яшчэ не)
2. Стварыць новы рэпазіторый:
   - Кнопка **New** → назва: `nowload-bot`
   - Выбраць **Private** (прыватны — ніхто не ўбачыць без запрашэння)
   - Без README (у нас ужо ёсць)
3. GitHub пакажа каманды — скапіруй блок «push an existing repository»

### Загрузіць файлы (зрабіць у тэрмінале, тэрчку раз):

```bash
cd /шлях/да/папкі/nowload     # перайсці ў папку бота
git init                       # ініцыялізаваць git
git add .                      # дадаць усе файлы
git commit -m "першы запуск"  # захаваць
git branch -M main
git remote add origin https://github.com/ТВА_ІМЯЮ/nowload-bot.git
git push -u origin main        # загрузіць на GitHub
```

### Важна — стварыць .gitignore (перад `git add .`):

Стварыць файл `.gitignore` у папцы бота з такім зместам:
```
google_creds.json
.env
__pycache__/
*.pyc
```

Гэта не дасць загрузіць сакрэтны ключ Google на GitHub.

### Як дадаваць змены пасля рэдагавання:

```bash
git add .
git commit -m "абнавіла тэкст у раздзеле скам"
git push
```

Усё. Тры каманды — і версія захавана.

## Крок 2 — Атрымаць токен бота ў Telegram

1. Адкрый [@BotFather](https://t.me/BotFather) у Telegram
2. Напіш `/newbot`
3. Задай імя (напр. `NowLoad`) і юзэрнейм (напр. `nowload_test_bot`)
4. Скапіруй токен — доўгі радок з лічбамі і літарамі
5. Захавай у бяспечным месцы (напрыклад у нататніку або ў Notion)

## Крок 3 — Запусціць бота лакальна (для тэставання)

Лакальны запуск — гэта калі бот працуе на тваім кампутары.  
Ідэальна для тэставання. Можна запускаць і спыняць у любы момант.

### Усталяваць Python і бібліятэкі (адзін раз):

Спампаваць Python з [python.org](https://python.org) калі не усталяваны.

```bash
pip install python-telegram-bot gspread google-auth
```

### Запісаць токен:

**Mac/Linux:**
```bash
export BOT_TOKEN="твой_токен_тут"
```

**Windows (у камандным радку):**
```
set BOT_TOKEN=твой_токен_тут
```

### Запусціць:

```bash
cd /шлях/да/папкі/nowload
python bot.py
```

Калі ўсё добра — убачыш: `NowLoad запушчаны ✅`

Каб спыніць: **Ctrl+C**

## Крок 4 — Хмарны хостынг (Railway) — для сталага запуску

Лакальны запуск гасне калі закрываеш ноўтбук.  
Для сталага запуску — Railway:

1. Зайдзі на [railway.app](https://railway.app)
2. «New Project» → «Deploy from GitHub repo» → выбраць `nowload-bot`
3. Дадаць зменную: `BOT_TOKEN` → твой токен (у раздзеле Variables)
4. Railway сам устанавіць залежнасці і запусціць бота

**Каштуе:** ~$5/месяц пасля бясплатнага ліміту.  
**Для бэта-тэставання:** лакальны запуск цалкам дастаткова.

## Крок 7 — Як рэдагаваць кантэнт (тэксты бота)

Усе тэксты — у файле **`content_poland.py`**.  
Структура кожнага экрана:

```python
"назва_раздзела": {
    "text": "Тут тэкст які бачыць карыстальнік",
    "buttons": [
        [("Тэкст кнопкі 1", "куды_вядзе")],
        [("Тэкст кнопкі 2", "куды_вядзе")],
    ]
},
```

### Каб змяніць тэкст:
1. Адкрыць `content_poland.py` у любым рэдактары (VSCode, Notepad++, нават Блокнот)
2. Знайсці раздзел па назве (Ctrl+F → увесці назву ключа, напр. `scam_check`)
3. Змяніць тэкст паміж лапкамі `"..."`
4. Захаваць файл
5. Перазапусціць бота (`python bot.py`)

### Правілы якія нельга парушаць:
- **Не выдаляць** `{MIN_WAGE_HOURLY_PL}` і падобнае — гэта зменныя лічбы з constants.py
- **Не мяняць** назвы ключоў (напр. `"scam_check"`) — бот не знойдзе раздзел
- **Не мяняць** `callback_data` ў кнопках (другое значэнне ў дужках) — зломяцца спасылкі

### Каб змяніць лічбы (зарплаты, ліміты):
Адкрыць **`constants.py`** — усе лічбы там.  
Змяніць значэнне — захаваць — перазапусціць бот.  
Больш нічога мяняць не трэба.

## Калі нешта зломіцца

**Бот маўчыць:**  
→ Праверы `echo $BOT_TOKEN` — ці ўсталяваны?  
→ Перазапусці `python bot.py`

**Памылка NameError / ImportError:**  
→ Паглядзі апошні радок у тэрмінале — там дакладна напісана дзе і ў чым памылка  
→ Звычайна гэта друкарская памылка ў content_poland.py

**Google Sheets не запісваецца:**  
→ Бот усё роўна працуе! Аналітыка проста прапускаецца.  
→ Праверы SPREADSHEET_ID у analytics.py
